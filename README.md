# استديو رفودي العراقي — موقع موسيقى متصل بـ MongoDB

موقع مكتبة أغاني تُقرأ مباشرة من قاعدة بيانات MongoDB، مع لوحة تحكم محمية برمز دخول لإضافة الأغاني،
ونظام "اطلب رفع أغنية" يسجّل فيه الزوار الدخول بحساب Google ويرسلون طلبات تحتاج موافقة الأدمن قبل نشرها.

## هيكل المشروع

```
music-games-site/
├── server.js                 # نقطة تشغيل السيرفر (Express)

├── middleware/
│   ├── adminAuth.js           # التحقق من صلاحية الأدمن (رمز الدخول + الجلسة)
│   └── userAuth.js             # التحقق من تسجيل دخول المستخدم عبر جوجل
├── models/
│   ├── Song.js                 # مخطط بيانات الأغنية
│   ├── SongRequest.js           # مخطط بيانات طلب رفع الأغنية
│   └── User.js                   # مخطط بيانات المستخدم المسجّل بجوجل
├── routes/
│   ├── songs.js                 # مسارات API الخاصة بالأغاني (CRUD)
│   ├── auth.js                   # مسارات تسجيل الدخول بجوجل
│   └── requests.js                # مسارات تقديم/موافقة/رفض طلبات رفع الأغاني

│   ├── css/style.css
│   └── js/

│       ├── admin.js                 # منطق الإضافة اليدوية + موافقة/رفض الطلبات
```

## خطوات التشغيل

### 1. تثبيت الاعتماديات
```bash
npm install
```

### 2. إعداد الاتصال بقاعدة بيانات MongoDB + رمز الأدمن
انسخ ملف `.env.example` باسم `.env`:

```bash
cp .env.example .env
```

وعدّل بداخله:
```
MONGODB_URI=mongodb+srv://username:password@cluster0.mongodb.net/music_games_db
ADMIN_CODE=الرمز-اللي-تختاره
ADMIN_SECRET=نص-عشوائي-طويل-وسري
```

- لو عندك MongoDB يعمل محلياً على جهازك، استخدم بدلاً منها:
  ```
  MONGODB_URI=mongodb://127.0.0.1:27017/music_games_db
  ```
- لو تستخدم MongoDB Atlas (سحابي مجاني)، أنشئ قاعدة من: https://www.mongodb.com/cloud/atlas

### 3. تشغيل السيرفر
```bash
npm start
```
أو أثناء التطوير (يعيد التشغيل تلقائياً عند أي تعديل):
```bash
npm run dev
```

بعد التشغيل، افتح المتصفح على:
```
http://localhost:3000
```

### 4. إعداد تسجيل الدخول بجوجل (لصفحة "اطلب رفع أغنية")

هذي الخطوة لازمة عشان يقدر الزوار يسجّلون دخول ويرسلون طلبات رفع أغاني:

1. روح إلى [Google Cloud Console](https://console.cloud.google.com/) → أنشئ مشروع جديد (أو استخدم مشروع موجود).
2. من القائمة الجانبية: **APIs & Services → OAuth consent screen** → اختر **External** واملأ اسم التطبيق وبريدك.
3. من **APIs & Services → Credentials** → **Create Credentials → OAuth client ID**.
4. نوع التطبيق: **Web application**.
5. في خانة **Authorized redirect URIs** ضيف بالضبط:
   ```
   http://localhost:3000/auth/google/callback
   ```
   (وإذا رفعت الموقع لسيرفر حقيقي، ضيف نفس الرابط لكن بدومينك الحقيقي بدل localhost، وحدّث `GOOGLE_CALLBACK_URL` بالـ `.env` بنفس القيمة).
6. بعد الإنشاء، انسخ **Client ID** و **Client Secret** وحطّهم بملف `.env`:
   ```
   GOOGLE_CLIENT_ID=...
   GOOGLE_CLIENT_SECRET=...
   GOOGLE_CALLBACK_URL=http://localhost:3000/auth/google/callback
   SESSION_SECRET=نص-عشوائي-طويل-وسري-مختلف-عن-ADMIN_SECRET
   ```
7. أعد تشغيل السيرفر بعد التعديل.

> بدون هذي الخطوة، صفحة `request-song.html` تظهر وزر "تسجيل الدخول بحساب Google" لكن الضغط عليه يرجّع خطأ من جوجل.

## لوحة التحكم (محمية برمز)

الوصول: `http://localhost:3000/admin.html`

الصفحة محمية بالكامل من طرف السيرفر — أي زيارة لها بدون تسجيل دخول صحيح تعرض صفحة طلب الرمز فقط (`admin-login.html`)، ولا تكشف أي محتوى من اللوحة الحقيقية. بعد إدخال `ADMIN_CODE` الصحيح، يُنشأ السيرفر جلسة موقّعة (كوكي) صالحة لمدة 7 أيام، وتقدر تسجّل خروج بالضغط على "تسجيل خروج" داخل اللوحة.

بمجرد الدخول، النموذج بسيط:
1. **عنوان الأغنية** (إجباري)
2. **اسم الفنان** (اختياري)
3. **رابط الأغنية المباشر** (إجباري — رابط ملف mp3 جاهز أونلاين)
4. **صورة الغلاف** (اختياري — تُرفع كملف صورة من جهازك)

بمجرد الضغط على **"نشر الأغنية على الموقع"**، الأغنية تُحفظ فوراً بقاعدة بيانات MongoDB، وتظهر **مباشرة** بمكتبة الموسيقى بالصفحة الرئيسية مع صورة الغلاف وزر التشغيل جاهزين.

> **مهم:** إضافة/تعديل/حذف الأغاني محمي أيضاً على مستوى الـ API نفسه (مو بس الواجهة) — يعني حتى محاولة استدعاء نقاط الإضافة مباشرة بدون تسجيل دخول كأدمن تُرفض تلقائياً برسالة 401. الزوار العاديون يقدرون فقط يسمعون الأغاني، بدون أي صلاحية إضافة.

## طلب رفع أغنية + الموافقة عليها من لوحة التحكم

### للزائر (صفحة `request-song.html`)
1. يضغط "اطلب رفع أغنية" من الصفحة الرئيسية.
2. يسجّل الدخول بحساب Google.
3. يعبّي عنوان الأغنية، الفنان (اختياري)، ثم إما رابط مباشر للأغنية أو رفع ملف صوتي من جهازه، وصورة غلاف اختيارية، وملاحظة اختيارية.
4. بعد الإرسال يشوف طلبه بقائمة "طلباتي" بحالة **قيد المراجعة**.

### للأدمن (تبويب "طلبات رفع الأغاني" داخل `admin.html`)
- يشوف كل الطلبات (قيد المراجعة / مقبولة / مرفوضة / الكل) مع معاينة صوتية مباشرة لكل طلب واسم/بريد صاحب الطلب.
- **قبول ونشر**: يحوّل الطلب فوراً لأغنية حقيقية تظهر بمكتبة الموسيقى بالصفحة الرئيسية.
- **رفض**: يقدر يكتب سبب اختياري يظهر للمستخدم بقائمة طلباته.

> نقاط الطلبات محمية بنفس نظامين الحماية الحاليين: تقديم الطلب يحتاج جلسة Google صالحة، والموافقة/الرفض يحتاج جلسة أدمن صالحة (نفس رمز `ADMIN_CODE`).

## ليش الصوت يستمر حتى لو تنقّلت بالصفحة؟
الموقع بُني كصفحة واحدة (Single Page)، وعنصر تشغيل الصوت (`<audio>`) موجود مرة واحدة فقط خارج قسم الأغاني، وما يُعاد إنشاؤه أبداً حتى لو تحدّثت القائمة. لذلك:
- التنقل داخل الصفحة لا يوقف الصوت أبداً.
- تبديل التبويبات أو تصغير المتصفح لا يوقفه أيضاً، طالما التبويب مفتوح.
- **الحالة الوحيدة اللي توقف الصوت**: إغلاق التبويب، أو عمل تحديث (Refresh/F5) للصفحة.

## نقاط الـ API المتاحة

| الطريقة | المسار              | الوظيفة                                    | يحتاج تسجيل دخول أدمن؟ |
|---------|---------------------|---------------------------------------------|--------------------------|
| GET     | /api/songs          | جلب كل الأغاني                              | لا                        |
| GET     | /api/songs/:id      | جلب أغنية واحدة                             | لا                        |
| POST    | /api/songs/upload   | إضافة أغنية (رابط أو ملف + صورة غلاف)       | نعم                       |
| POST    | /api/songs          | إضافة أغنية جديدة عبر JSON                  | نعم                       |
| PUT     | /api/songs/:id      | تعديل أغنية                                 | نعم                       |
| DELETE  | /api/songs/:id      | حذف أغنية                                   | نعم                       |
| POST    | /api/admin/login    | تسجيل الدخول للوحة الأدمن (بالرمز)          | —                         |
| POST    | /api/admin/logout   | تسجيل الخروج من لوحة الأدمن                 | —                         |
| GET     | /auth/google        | بدء تسجيل الدخول بجوجل                       | —                         |
| GET     | /auth/google/callback | رابط رجوع جوجل بعد الموافقة                | —                         |
| GET     | /auth/me            | معرفة هوية المستخدم الحالي (أو null)         | —                         |
| POST    | /auth/logout        | تسجيل خروج المستخدم من حسابه بجوجل           | يحتاج جلسة مستخدم         |
| POST    | /api/requests        | تقديم طلب رفع أغنية جديد                    | يحتاج جلسة مستخدم (Google) |
| GET     | /api/requests/mine    | جلب طلبات المستخدم الحالي فقط                | يحتاج جلسة مستخدم (Google) |
| GET     | /api/requests         | جلب كل الطلبات (فلترة بـ `?status=`)         | نعم (أدمن)                |
| POST    | /api/requests/:id/approve | قبول طلب ونشره كأغنية فعلية              | نعم (أدمن)                |
| POST    | /api/requests/:id/reject  | رفض طلب (مع سبب اختياري)                 | نعم (أدمن)                |
| DELETE  | /api/requests/:id     | حذف طلب نهائياً                              | نعم (أدمن)                |

## ملاحظة مهمة
هذا المشروع يحتاج بيئة Node.js حقيقية (على جهازك أو خادم استضافة) لتشغيل السيرفر والاتصال الفعلي بـ MongoDB — فهو لا يعمل داخل نافذة الدردشة نفسها. الملفات جاهزة بالكامل؛ ما عليك سوى تشغيل الخطوات أعلاه.
